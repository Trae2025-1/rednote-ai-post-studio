# Rednote AI Post Studio

从一个主题词开始，一次生成小红书笔记的标题、正文、关键词和封面图。

`rednote-ai-post-studio` 是一个面向 AI 工具 / 办公效率 / Agent 实战账号的 Codex Skill。它不是单纯写文案，也不是单纯做封面，而是把小红书发布前的关键环节打成一条稳定流程：

```text
主题词
→ 选题定位
→ 标题候选
→ 推荐标题
→ 正文文案
→ 关键词标签
→ 封面文案
→ 参考库驱动封面图
→ PNG 预览
→ 合规检查
→ 发布建议
```

默认会参考我本人账号「梓丰学AI中」的内容风格：普通人学 Agent / Codex / AI 办公实战，真实复盘，不装专家，不强营销。

## 适合谁用

- 正在做小红书 AI 工具号、AI 办公号、效率工具号的人。
- 想持续输出 Codex、ChatGPT、DeepSeek、NotebookLM、飞书、PPT、Excel、网页设计等内容的人。
- 想让笔记从「想到一个主题」直接推进到「可发布文案 + 可用封面」的人。
- 不想每次都重新想标题、正文结构、标签和封面排版的人。

## 核心优势

### 1. 从主题词直接到发布包

只需要给一个主题，例如：

```text
DeepSeek 办公实测
```

Skill 会自动补齐：

- 内容定位
- 标题候选
- 推荐标题
- 正文文案
- 标签关键词
- 封面文案
- 发布建议

### 2. 文案不是泛泛 AI 味，而是实战复盘风格

默认正文结构贴近真实账号表达：

```text
这次我测试了什么
以前这个任务卡在哪
我怎么拆流程
用了哪些工具 / Skill
实际效果如何
适合哪些场景
一句结论
```

它更适合做「我真的试过」的内容，而不是空泛的 AI 口号。

### 3. 标题更适合 AI 办公实战号

内置标题方向包括：

```text
我用【工具】做了个【可复用能力】
用【工具/智能体】搞定【具体任务】
不会【技能】？Codex直接帮你【结果】
【工具A】和【工具B】，普通人先学哪个？
DayX｜【工具】办公实战
```

目标是让标题有工具名、有任务、有结果感，但不夸张。

### 4. 关键词按小红书发布场景组织

关键词会按这些维度生成：

- 工具词：Codex、ChatGPT、DeepSeek、NotebookLM、Skill、智能体
- 场景词：AI办公、PPT制作、Excel表格、封面设计、网页设计
- 能力词：AI工作流、AI提效、自动化办公、可编辑设计
- 人群词：普通人学AI、职场人、打工人AI实验室
- 平台词：小红书封面、小红书图文、小红书标题

如果你要求「联网找近期关键词」或「查热门搜索词」，Skill 会先调研再生成。

### 5. 封面以参考库为强依据，不再机械套模板

封面生成会先从内置参考图里选择最接近的视觉类型，再按照这张参考图的版式、留白、标题层级、logo 位置、底部结构和色调规则来做。

新的封面流程不是简单把 SVG 模板换字，也不是先生成背景再手工排版，而是：

```text
对比 2-3 种参考风格
→ 选定一张具体参考图
→ 提取 logo / 标语 / 标题 / CTA 替换清单
→ 调用 image 直接生成同款成品封面
→ 只替换真实 logo 和标语/标题/文案
→ 可在参考风格内调整色调，让封面更好看、更有区分
→ 必要时只做精确文字或 logo 修正
→ 按清单验收
```

这样既能保持账号统一感，也能让封面更像成品小红书头图，而不是工程模板截图。

一句话：Skill 不当设计师，不用前端/Pillow/SVG 手搓封面；它只选参考母版、写 image 生成指令、检查结果是否真的像参考图。

重点：参考图就是审美依据，不是灵感板。参考图里没有的蓝线、弧线、箭头、下划线、贴纸、徽章、图标、分割线、装饰块，一律不能加。

同时也不能把参考图过度简化。参考里如果有角标、强调线、气泡框、点阵、底部弧形色块、装饰短线，就要让 image 按同类元素保留下来，只是替换 logo、标题文案和配色。

字体和排布也必须锁死参考图：主标题、产品词、底部 CTA、小标签都要复制参考图的字号层级、粗细、对齐、间距和胶囊/色块处理。不能把底部小字当普通文本打一行，也不能把产品名硬塞进中文短语中间，例如不能把「我的首席技术官 Cursor」排成「我的首席 / Cursor / 技术官」。

Logo 也必须锁死参考图的替换方式：参考图哪里是 logo，就用用户上传或查到的真实 logo 直接替换到那个位置。不能让 image 生成一个涂抹、马赛克、模糊、粗糙画板式的 logo 背板来糊弄。

允许改色调，但必须是为了更美观、更有区分度，并且由 image 在参考图风格内完成。Logo 色调只能作为配色线索，不是整张海报必须完全同色的命令。不能用粗糙色块、随机线条、前端卡片感来假装“换风格”。

参考图锁的是版式、层级、元素密度和装饰类型，不是锁死配色。新封面不能和参考图色调一模一样，必须通过 image 做出有审美的色调区分。

配色也不能用 SVG 或手工后期去改。要在 image 生成指令里说清楚色调方向，让 image 一次性生成接近成品的海报。

Skill 应该主动判断配色，不需要用户手动提醒。遇到黑白 logo、过饱和 logo、太冷/太暗的 logo，或者连续多张封面容易同质化时，要主动尝试 2-3 种色调方向，比如「logo 点缀 + 中性底」「冷调高级」「暖白编辑感」「清爽蓝绿」「高对比黑白 + 少量强调色」，再选择最适合参考母版的一版。

菜单/利益点必须做细节验收：对号和文字要在同一视觉高度，左右间距要跟参考图一致；如果副标题已经写了「写代码 / 改 Bug / 做方案」，下面的小菜单就不能再重复这三个词。

## 快速使用

在 Codex 中调用：

```text
Use $rednote-ai-post-studio to create a Xiaohongshu post and cover from this topic: DeepSeek 办公实测
```

中文也可以：

```text
用 rednote-ai-post-studio，帮我做一篇小红书，主题是 DeepSeek 办公实测，直接出标题、正文、关键词和封面。
```

输出会包含：

```text
rednote-ai-post-studio-output/<topic-slug>/publish-pack.md
rednote-ai-post-studio-output/<topic-slug>/cover.svg 或其他可编辑源/叠加文件
rednote-ai-post-studio-output/<topic-slug>/cover.png
```

## 推荐输入方式

最简单输入：

```text
主题：DeepSeek 办公实测
```

更高质量输入：

```text
主题：用 Codex 做小红书封面
目标人群：刚开始学 AI 的职场人
内容类型：实战复盘
重点：不是一次性出图，而是可复用封面流程
封面关键词：Codex、可编辑、审美在线
```

系列内容输入：

```text
主题：7天 Codex 办公实战 Day 3
任务：用 Codex 整理 Excel 表格
风格：普通人实测，不要课程感
```

带 logo 的封面输入：

```text
主题：NotebookLM 资料整理教程
Logo：/path/to/notebooklm-logo.svg
封面要求：使用参考库里最接近的单工具教程封面，调用 image 生成同款成品封面，只替换 logo 和标题文案，可以在参考风格内调整色调。
```

## 封面生成硬规则

封面不是自由重设计，必须严格遵循参考库里的图和模板。

### 允许

- 使用参考库中的具体案例作为强视觉依据。
- 生成前对比 2-3 种参考风格，并选定一张具体参考图。
- 根据用户选择的标题或推荐标题确定封面主标题。
- 调用 image 能力直接生成符合参考图的成品封面。
- 只替换参考图里的 logo 和标语/标题/文案。
- 在原有参考图风格内调整背景色、强调色或整体色调，让封面更美观、更有区分。
- 根据 logo 色调选择局部强调色，但整体配色由海报审美和参考母版决定。
- 主动尝试不同色调搭配，不等用户提醒换色。
- 保留参考图版式，但不能照搬参考图色调。
- 保留参考图的信息密度和海报层次，不能把丰富参考图做成过于简单的空白海报。
- 根据用户上传的 logo 放置真实 logo。
- 如果用户要求联网找公开 logo，并且环境允许联网，可以查找真实 logo 资产后放入对应位置。
- 如 image 生成的文字或 logo 不够精确，只允许在同一位置做最小修正，不能重新设计排版。
- 底部小字必须做成参考图里的 CTA、胶囊、色块或标签样式，不能像普通打字。
- 只允许使用选定参考图里已经出现的装饰元素。
- 直接用真实 logo 替换参考图 logo 区，不生成粗糙占位背板。
- 对号/图标和文字必须垂直居中对齐。
- 副标题、小菜单、CTA 三层文案不能互相重复。

### 禁止

- 更换海报字体。
- 改变海报版式结构。
- 添加新的装饰风格。
- 重绘、改色、裁切或重新设计 logo。
- 没有真实 logo 时凭空发明 logo。
- 脱离参考库重新设计一张新海报。
- 用前端、Pillow、SVG 或手工坐标自己设计封面。
- 先生成空背景，再手工摆 logo、标题、CTA。
- 用 SVG 或手工后期补元素、改配色、加装饰。
- 直接采用 image 模型生成的变形中文作为最终封面标题。
- 让 image 模型凭空复刻品牌 logo。
- 自己额外添加参考图里没有的蓝线、弧线、箭头、下划线、分割线、贴纸、徽章、图标或装饰块。
- 以“更好看”“更有动感”为理由自由发挥。
- 用粗糙色块或随机装饰假装色调优化。
- 使用默认小字号文本糊在底部 CTA 或标签区域。
- 把产品名插进中文短语中间，破坏标题语义和阅读顺序。
- 用涂抹、马赛克、模糊块、粗糙画板当 logo 背板。
- 对号和文字高度不一致。
- 副标题和小菜单重复表达同一件事。

## 生成质量检查

封面生成后会按清单校验：

- 是否选定了具体参考图或参考类型。
- 是否对比过不同参考风格，并说明最终选择。
- 是否沿用了参考图的版式、标题层级、logo 位置和装饰风格。
- 是否沿用了参考图的字体粗细、字号层级、行距、对齐和底部 CTA 样式。
- 是否由 image 直接生成成品封面，而不是手工前端式排版。
- 是否只替换 logo 和标语/标题/文案。
- 是否保留了参考图的信息密度和海报层次，没有过度简化。
- 标题是否读得顺，产品名是否没有破坏中文短语。
- 是否存在参考图里没有的多余装饰；如果有，必须删除或重做。
- logo 是否直接替换到参考图 logo 区，而不是坐在粗糙占位背板上。
- 对号/图标和文字是否视觉居中对齐。
- 副标题、小菜单、CTA 是否各自承担不同信息。
- 是否使用用户选择的标题或推荐标题。
- logo 是否来自用户上传或真实查找，而不是模型重画。
- 中文标题是否清晰可读，没有变形。
- 色调调整是否更美观、更有区分，并且仍然属于参考图风格。
- 是否把 logo 色调当作局部线索，而不是让整张海报完全跟 logo 同色。
- 是否在需要时主动比较过不同色调方向。
- 是否没有照搬参考图原配色，而是做了清晰的色调区分。

如果输出 SVG 或 SVG 叠加层，还会用脚本校验：

```bash
node scripts/validate_cover.js rednote-ai-post-studio-output/<topic>/cover.svg rednote-ai-post-studio-output/<topic>/cover.png
```

校验重点包括：

- 是否是 3:4 小红书竖版比例。
- SVG 是否保留可编辑文本。
- PNG 是否成功生成。
- 是否存在未替换占位符。
- 是否满足基础交付条件。
