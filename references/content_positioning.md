# Content Positioning

## Topic Classes

Choose one class for each post.

### Real-Test Recap

Use when the user tried a tool, Skill, or workflow.

Pattern:

- What I tested.
- What used to be hard.
- How I broke it into steps.
- What worked.
- What still needs tweaking.
- Who it is suitable for.

### Workflow Tutorial

Use when the user wants a how-to.

Pattern:

- Suitable reader.
- Preparation.
- Steps.
- Tool responsibilities.
- Output.
- Reusable lesson.

### Tool Comparison

Use for ChatGPT vs Codex, Codex vs other tools, or "普通人先学哪个".

Pattern:

- Why compare.
- Use case differences.
- Learning cost.
- Output type.
- Recommendation by user type.

### Pitfall Note

Use when the hook is failure, misunderstanding, or avoiding waste.

Pattern:

- Common mistake.
- Why it happens.
- How to fix it.
- Better workflow.

### Series Day Post

Use for Day1-Day30 and repeated practice.

Pattern:

- Day marker.
- One concrete task.
- One workflow discovery.
- One next step.

### Reusable Skill/Template Post

Use when the output is a Skill, template, workflow, or reusable asset.

Pattern:

- The repeated task.
- Why prompt-only was not enough.
- What the Skill/template controls.
- Example output.
- Why reuse matters.

## Default Content Direction

If the user gives only a topic, default to Real-Test Recap.

If the topic includes "Day", "系列", "第几天", use Series Day Post.

If the topic includes "怎么", "教程", "步骤", use Workflow Tutorial.

If the topic includes "对比", "哪个", "怎么选", use Tool Comparison.

If the topic includes "坑", "问题", "翻车", use Pitfall Note.
