# Copywriting Frameworks

## Default: Practical Test Recap

Use this most often.

Structure:

1. This time I used Codex/Agent/Skill to do a concrete task.
2. This task used to get stuck at several points.
3. I broke the workflow into steps.
4. I used these tools or Skills.
5. Each tool handled a specific responsibility.
6. Actual result or biggest feeling.
7. Suitable scenarios.
8. One grounded conclusion.

Useful sentence patterns:

- 这次不是单纯让 AI 出一个结果，而是把我常用的流程整理成了可复用规则。
- 以前做这个，经常卡在几个点：
- 这次我重点测试了：
- 对我来说，最大的价值不是做一次，而是以后可以反复复用。
- 适合用在：
- 结论：

## Workflow Tutorial

Use when the user asks for steps.

Structure:

1. Suitable reader.
2. Goal.
3. Preparation.
4. Step-by-step process.
5. Common checks.
6. Output.
7. Reuse suggestion.

## Tool Comparison

Structure:

1. Why compare.
2. Tool A is better at...
3. Tool B is better at...
4. Ordinary users should choose by task.
5. Practical recommendation.

## Pitfall Note

Structure:

1. The mistake.
2. Why it causes bad output.
3. Better prompt/workflow.
4. What to check.
5. Safer conclusion.

## Series Day Post

Structure:

1. Day marker.
2. Today's task.
3. What I tried.
4. What surprised me.
5. What can be reused.
6. What I will test next.

## Paragraph Style

- Keep paragraphs short.
- Prefer one idea per paragraph.
- Use bullets for "卡点", "用到的 Skill", "适合用在".
- Do not write long motivational intros.
- Do not use excessive emojis.
