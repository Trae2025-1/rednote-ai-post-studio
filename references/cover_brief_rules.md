# Cover Brief Rules

## Purpose

Turn the publishing package into a strict cover-edit brief.

## Required Fields

- Selected reference/template:
- Ratio: 3:4
- Main title:
- Subtitle:
- Corner label:
- Bottom note:
- Logo source:
- Logo decision:
- Pure-text/no-logo treatment:
- Background/accent color adjustment:
- Locked elements:
- Candidate reference styles:
- Final selected reference image:
- Reference typography to copy:
- Final title grouping:
- Product/tool name placement:
- CTA/bottom text treatment:
- Logo slot replacement:
- Icon-label row alignment:
- Copy de-duplication plan:
- Allowed decorations from reference:
- Forbidden extra decorations:
- Replacement map:
- Image-generation instruction:
- Optional color-tone adjustment:
- Candidate color-tone directions:
- Chosen color-tone direction:
- Logo-color usage:
- Reference-palette differentiation:
- Manual post-processing limit:
- Production mode: image-generated finished poster from reference, with minimal exact correction only if needed
- Output: final PNG + generated composite + minimal correction overlay when applicable

## Locked Elements

Always state:

- Preserve poster font.
- Preserve poster font weight, line spacing, alignment, and title hierarchy.
- Preserve layout.
- Preserve decoration style.
- Preserve hierarchy.
- Preserve CTA/chip/band treatment for bottom copy.
- Preserve the user's title semantics; do not split connected phrases with the tool name.
- Replace the reference logo slot directly with the real logo; no rough placeholder or mosaic backplate.
- Align check/menu icons and text on the same visual center.
- Avoid repeating subtitle copy in the menu/benefit row.
- Only replace logo and slogan/title/copy.
- Color tone may be adjusted through image generation to make the cover more beautiful and differentiated while preserving the reference style.
- Logo color is only a palette cue; the whole poster does not need to match the logo exactly.
- Proactively consider 2-3 tone directions when a different palette could improve the result.
- Choose the color direction for poster quality, readability, and differentiation, not literal logo matching.
- Preserve the reference layout, but do not copy the reference palette exactly.
- Use image generation to create the finished poster, not just a background layer.
- Manual post-processing may only correct exact logo/text in the same slots.
- Do not use frontend/Pillow/SVG/manual layout as the cover design method.
- Do not generate a blank background and manually arrange the poster afterward.
- Do not add any decoration that is not visible in the selected reference image.
- No extra blue lines, curved strokes, arrows, underlines, separators, badges, icons, or blobs unless copied from the selected reference.
- No default-looking small text in the bottom CTA or benefit area.
- No misaligned check/menu labels.
- No duplicate subtitle/menu copy.

## No Logo Available

If no logo is provided and no lookup is requested or available:

- For public, well-known product/tool names, use web lookup for an official or reliable logo when available; mention that user upload is more precise if they have a preferred version.
- For private, personal, internal, ambiguous, or uncertain brands, recommend that the user upload a logo instead of guessing.
- If the user wants a pure-text cover, choose a no-logo-friendly reference or convert the logo slot into a designed text focal point.
- Pure-text logo-slot conversions may use a large keyword, chapter number, topic label, or core value card.
- Do not leave an empty logo hole.
- Do not invent a logo.
